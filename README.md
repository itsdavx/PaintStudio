# PaintStudio

**Interactive Digital Drawing Application — ESPE 2026**

PaintStudio es una aplicación gráfica interactiva inspirada en Paint, desarrollada como proyecto final para la asignatura de **Computación Gráfica** en la **Universidad de las Fuerzas Armadas ESPE**.

El proyecto tiene como objetivo aplicar conceptos fundamentales de computación gráfica, programación orientada a objetos y diseño de interfaces gráficas mediante el desarrollo de una herramienta de dibujo digital.

---

## Objetivos del Proyecto

* Implementar una aplicación de dibujo interactiva tipo Paint.
* Aplicar conceptos de Computación Gráfica.
* Diseñar una arquitectura modular basada en POO.
* Implementar algoritmos gráficos manualmente.
* Desarrollar una interfaz intuitiva y funcional.

---

## Características Principales

* Lienzo de dibujo interactivo
* Dibujo de múltiples figuras geométricas
* Herramienta de selección
* Selección de color de línea y relleno
* Control de grosor de trazo
* Herramienta de borrado
* Limpieza completa del lienzo
* Transformaciones geométricas

  * Traslación
  * Rotación
  * Escalado
* Algoritmos de relleno de figuras
* Gestión de eventos del mouse
* Guardado del proyecto
* Exportación como imagen

---

## Conceptos de Computación Gráfica Aplicados

Este proyecto implementa conceptos fundamentales de computación gráfica:

* Sistema de coordenadas cartesianas
* Rasterización de figuras
* Manipulación de píxeles
* Renderizado en tiempo real
* Algoritmos de relleno
* Transformaciones geométricas
* Gestión de eventos interactivos
* Actualización dinámica del canvas

---

## Arquitectura del Proyecto

El sistema está diseñado bajo el patrón arquitectónico **MVC (Model-View-Controller)**, permitiendo una separación clara entre la lógica de negocio, la interfaz gráfica y el control de eventos.

* **Model:** Gestiona los datos y estructuras relacionadas con las figuras y elementos gráficos.
* **View:** Representa la interfaz gráfica y el lienzo interactivo.
* **Controller:** Administra la interacción del usuario mediante eventos del mouse y acciones del sistema.

Esta arquitectura facilita la modularidad, escalabilidad y mantenimiento del sistema, permitiendo una mejor organización del código y una separación adecuada de responsabilidades.
